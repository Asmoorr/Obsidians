## Задание 1. Свойства сетевого подключения

Для просмотра свойств сетевого подключения откроем список сетевых адаптеров в панели управления с помощью команды `ncpa.cpl`.

> 📷 Рисунок 1 — Сетевые подключения

### Назначение компонентов

**Клиент для сетей Microsoft**
Позволяет компьютеру получать доступ к общим ресурсам сети. Благодаря этому компоненту можно находить папки на других компьютерах и использовать сетевые принтеры.

**Служба доступа к файлам и принтерам сетей Microsoft**
Позволяет другим компьютерам в сети получать доступ к ресурсам вашего компьютера. Благодаря этой службе есть возможность поделиться папкой на своём ПК, чтобы другой человек мог скачать оттуда файл.

**IP версии 4 (TCP/IPv4)**
Базовый сетевой протокол, обеспечивающий адресацию и маршрутизацию данных в сети. IP отвечает за уникальный адрес компьютера, TCP — за доставку данных. Является основным компонентом взаимодействия компьютера в сети.

## Задание 3. Утилита ping

**ping** (Packet InterNet Groper) работает на сетевом уровне модели OSI, базируется на протоколе ICMP. Используется для проверки доступности устройства в сети и измерения времени отклика.

### Стандартное использование

```
ping fpin.itmo.ru
```

> 📷 Рисунок 3 — Стандартное использование ping

По умолчанию отправляется 4 ICMP Echo-Request с размером данных 32 байта. Сначала выполняется DNS-разрешение имени, затем последовательно отправляются пакеты. В выводе: IP-адрес узла, время задержки (мс) и TTL (Time to Live — максимальное количество маршрутизаторов до уничтожения пакета).

### Ключ `-t` — непрерывная отправка

```
ping -t 8.8.8.8
```

> 📷 Рисунок 4 — Использование ping с флагом -t

Переводит утилиту в режим непрерывной отправки ICMP-запросов. Полезен для наблюдения за стабильностью соединения в реальном времени. Остановка — `Ctrl+C`.

### Ключ `-n` — количество пакетов

```
ping -n 10 8.8.8.8
```

> 📷 Рисунок 5 — Использование ping с флагом -n

Переопределяет стандартное количество отправляемых запросов (4) и задаёт точное число пакетов.

### Ключ `-l` — размер пакета

```
ping -l 1000 8.8.8.8
```

> 📷 Рисунок 6 — ping с флагом -l

Задаёт размер поля данных в отправляемом ICMP-пакете. При значении 10000 байт пакет может не доходить до целевого хоста из-за ограничений MTU.

### Перенаправление вывода

```
ping 8.8.8.8 > C:\ping_results.txt
```

> 📷 Рисунок 7 — ping с перенаправлением вывода
> 📷 Рисунок 8 — Результат перенаправления вывода

Оператор `>` перенаправляет STDOUT в файл вместо консоли. Если файл уже существует — содержимое перезаписывается.

## Задание 4. Утилита tracert

**tracert** (Trace Route) — диагностический инструмент сетевого уровня, показывающий полный путь IP-пакета от компьютера до конечного узла через цепочку маршрутизаторов.

**Принцип работы:** манипуляция полем TTL в заголовке IP-пакета.

1. Отправляется серия из 3 ICMP-запросов с TTL = 1
2. Первый маршрутизатор уменьшает TTL до 0 и отправляет ICMP Time Exceeded
3. tracert фиксирует IP-адрес и время отклика первого узла
4. TTL увеличивается на 1, процесс повторяется до достижения цели

```
tracert amazingbread.com
```

> 📷 Рисунок 9 — Базовое использование tracert

«Звёздочки» `* * *` означают, что маршрутизатор игнорирует ICMP-пакеты (настроен в целях безопасности или снижения нагрузки).

```
tracert -h 15 -w 500 amazingbread.com
```

> 📷 Рисунок 10 — Использование tracert с флагами -h и -w

- `-h` — максимальное количество прыжков (по умолчанию 30)
- `-w` — таймаут ожидания ответа от каждого узла в мс

## Задание 5. Утилиты ipconfig и net

### ipconfig

**ipconfig** предназначена для отображения параметров TCP/IP, управления кэшем DNS и обновления аренды IP-адресов от DHCP-сервера.

```
ipconfig /all
```

> 📷 Рисунок 11 — Вывод информации о сетевых адаптерах

Ключевые поля вывода:

| Поле | Значение |
|------|----------|
| DHCP включён | Да/Нет — режим получения IP |
| IPv4-адрес | Текущий адрес в локальной сети |
| Маска подсети | Определяет границу сети |
| Основной шлюз | IP роутера для выхода в интернет |
| Физический адрес | MAC-адрес сетевой карты |
| DNS-суффикс | Домен, добавляемый к коротким именам хостов |
| Аренда получена / истекает | Срок действия IP, выданного DHCP |

```
ipconfig /release    # освободить IP-адрес
ipconfig /renew      # запросить новый адрес у DHCP
ipconfig /flushdns   # очистить кэш DNS
ipconfig /displaydns # показать кэшированные записи DNS
```

> 📷 Рисунок 12 — Вывод кэша DNS

Комплекс утилит для управления ресурсами SMB, сессиями и учётными записями.


## Задание 6. Batch-скрипт настройки сетевого адаптера

Разработан исполняемый batch-файл `lab1-batch-configure-network.bat`. Скрипт использует утилиту `netsh` для управления сетевым стеком Windows.

В начале файла устанавливается кодировка `chcp 1251` — необходима для корректного отображения кириллицы в выводе `netsh` и `ipconfig` на русскоязычной Windows.

Поскольку CMD не поддерживает работу с объектами, для автоматического определения активного беспроводного адаптера используется однострочный вызов PowerShell через конструкцию `for /f`. Команда получает числовой индекс адаптера путём фильтрации по признаку `PhysicalMediaType -like '*802.11*'` и статусу `Up`, сохраняет его в переменную `%IF%`. Использование индекса вместо имени позволяет избежать проблем с передачей кириллицы внутри CMD.

```
1 - DHCP
2 - Static (192.168.1.50 / 255.255.255.0 / gw 192.168.1.1)
```

**Пункт 1 — DHCP:**
```bat
netsh interface ipv4 set address name=%IF% source=dhcp
netsh interface ipv4 set dns    name=%IF% source=dhcp
```
`source=dhcp` переводит интерфейс в режим автоматического получения IP и DNS от DHCP-сервера.

**Пункт 2 — Static:**
```bat
netsh interface ipv4 set address name=%IF% static 192.168.1.50 255.255.255.0 192.168.1.1
netsh interface ipv4 set dns    name=%IF% static 8.8.8.8
```
Устанавливает статический IP, маску и шлюз. DNS — публичный сервер Google (8.8.8.8). При подключении к другой сети доступ в интернет прекратится, что подтверждает корректную работу скрипта.

## Задание 7. PowerShell-скрипт настройки сетевого адаптера

Разработан скрипт `lab1-powershell-configure-network.ps1`. В отличие от batch-версии, PowerShell предоставляет доступ к объектной модели Windows и нативно поддерживает кириллицу — ручное управление кодировкой не требуется.

**Проверка прав администратора:**
```powershell
$isAdmin = ([Security.Principal.WindowsPrincipal][Security.Principal.WindowsIdentity]::GetCurrent())
              .IsInRole([Security.Principal.WindowsBuiltInRole]::Administrator)
```
Настройка сетевых параметров требует прав администратора. При их отсутствии скрипт завершается с предупреждением.

**Определение адаптера через `Get-NetAdapter`:**
```powershell
$adapter = Get-NetAdapter | Where-Object { $_.InterfaceType -eq 71 -and $_.Status -eq 'Up' } | Select-Object -First 1
```
Фильтрация по `InterfaceType -eq 71` (стандарт IEEE 802.11, Wi-Fi) и статусу `Up`. Извлекается имя адаптера — PowerShell корректно передаёт кириллицу в команды `netsh`. Если адаптер не найден, выводится таблица всех интерфейсов через `Get-NetAdapter | Format-Table`.

**Меню скрипта:**
```
Adapter: Беспроводная сеть

1 - DHCP
2 - Static (192.168.1.50 / 255.255.255.0 / gw 192.168.1.1 / dns 8.8.8.8)
3 - Adapter info (model, link, speed, duplex)
```

**Пункты 1 и 2** — настройка через `netsh` с подстановкой имени адаптера:
```powershell
netsh interface ipv4 set address name="$($adapter.Name)" source=dhcp
netsh interface ipv4 set dns    name="$($adapter.Name)" source=dhcp
```

**Пункт 3 — информация об адаптере:**
```powershell
Write-Host "Model:       $($adapter.InterfaceDescription)"
Write-Host "Link:        $($adapter.Status)"
Write-Host "Speed:       $($adapter.LinkSpeed)"
Write-Host "Full Duplex: $($adapter.FullDuplex)"
```
- `InterfaceDescription` — полное название модели сетевой карты
- `Status` — наличие физического подключения (Up / Down)
- `LinkSpeed` — текущая согласованная скорость (например, 433.3 Mbps)
- `FullDuplex` — True означает полнодуплексный режим

`net use` подключает сетевую папку или принтер как локальный ресурс — можно обращаться по букве диска без указания UNC-пути каждый раз.


**6. Режимы дуплекса**

**Half Duplex (полудуплекс)** — передача возможна только в одном направлении одновременно. Принцип рации. Возможны коллизии. Характерен для старых сетей Ethernet на общей шине (CSMA/CD).

**Full Duplex (полный дуплекс)** — передача и приём происходят одновременно и независимо. Коллизий нет. Используется во всех современных коммутируемых сетях. При 100 Mbps — реально 100 Mbps в каждую сторону одновременно.

**Auto-Duplex** — адаптеры автоматически согласовывают режим по стандарту IEEE 802.3u. Рекомендуется по умолчанию.

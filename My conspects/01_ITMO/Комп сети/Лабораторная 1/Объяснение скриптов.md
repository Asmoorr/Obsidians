# Объяснение скриптов

> **Лабораторная работа №1** | Компьютерные сети | 2026-03-24

---

## Краткая теория

| Понятие | Описание |
|---------|----------|
| **IP-адрес** | Числовой идентификатор устройства в сети |
| **Маска подсети** | Определяет, какая часть IP — сеть, а какая — хост (`255.255.255.0` = /24) |
| **Шлюз** | IP маршрутизатора, через который идёт трафик наружу |
| **DNS** | Сервер перевода доменных имён в IP (`8.8.8.8` — Google DNS) |
| **DHCP** | Протокол автоматической выдачи IP-настроек |
| **netsh** | Утилита Windows для настройки сетевого стека из CMD |
| **Get-NetAdapter** | PowerShell-командлет для информации об адаптере |

## Скрипт 1: `lab1-batch-configure-network.bat` (Задание 6)

Batch-скрипт для CMD. Использует утилиту `netsh`.

### Полный код с объяснениями

```bat
@echo off
chcp 1251 >nul
```
`chcp 1251` — переключает консоль в кодировку Windows-1251 (кириллица).
Без этой строки вывод netsh/ipconfig на русской Windows отображался бы кракозябрами.

```bat
for /f %%i in ('powershell -NoProfile -Command "Get-NetAdapter -Physical | Where-Object { $_.PhysicalMediaType -like '*802.11*' -and $_.Status -eq 'Up' } | Select-Object -First 1 -ExpandProperty InterfaceIndex"') do set "IF=%%i"

if not defined IF (
    echo Active Wi-Fi adapter not found.
    netsh interface show interface
    pause
    exit /b
)

echo Adapter index: %IF%
```
CMD не умеет работать с объектами, поэтому для автодетекта вызывается PowerShell одной командой.
`for /f` перехватывает вывод команды в скобках и записывает его в переменную `%%i` → `%IF%`.
PowerShell возвращает только числовой индекс — нет проблем с кириллицей в CMD.
Если адаптер не найден, показывается список интерфейсов через `netsh interface show interface`.

```bat
echo 1 - DHCP
echo 2 - Static (192.168.1.50 / 255.255.255.0 / gw 192.168.1.1)
set /p c="> "
```
Простое меню. `set /p` считывает ввод пользователя в переменную `c`.

```bat
if "%c%"=="1" (
    netsh interface ipv4 set address name=%IF% source=dhcp
    netsh interface ipv4 set dns    name=%IF% source=dhcp
)
```
Переключение на DHCP:
- `set address ... source=dhcp` — IP-адрес будет получен автоматически от DHCP-сервера
- `set dns ... source=dhcp` — DNS тоже автоматически
- `name=%IF%` — здесь `%IF%` число (индекс), netsh принимает и имя, и числовой индекс

```bat
if "%c%"=="2" (
    netsh interface ipv4 set address name=%IF% static 192.168.1.50 255.255.255.0 192.168.1.1
    netsh interface ipv4 set dns    name=%IF% static 8.8.8.8
)
```
Установка статического IP: адрес, маска, шлюз — в одной строке. DNS — отдельно.
Статический IP работает только если устройство подключено к сети `192.168.1.x`.

## Скрипт 2: `lab1-powershell-configure-network.ps1` (Задание 7)

PowerShell-скрипт. Для настройки IP использует `netsh` (как и bat), а для поиска адаптера — `Get-NetAdapter`.

### Полный код с объяснениями

```powershell
$isAdmin = ([Security.Principal.WindowsPrincipal][Security.Principal.WindowsIdentity]::GetCurrent())
              .IsInRole([Security.Principal.WindowsBuiltInRole]::Administrator)
if (-not $isAdmin) { ... exit }
```
Проверка прав администратора. Настройка сети требует admin-прав.
`WindowsIdentity::GetCurrent()` — текущий пользователь. `IsInRole(Administrator)` — проверка группы.

```powershell
$adapter = Get-NetAdapter | Where-Object { $_.InterfaceType -eq 71 -and $_.Status -eq 'Up' } | Select-Object -First 1
if (-not $adapter) {
    Write-Host "No active Wi-Fi adapter found:"
    Get-NetAdapter | Format-Table Name, InterfaceIndex, Status
    Read-Host "Press Enter to exit"
    exit
}
```
Автоматически находит активный Wi-Fi адаптер.
`InterfaceType -eq 71` — тип интерфейса IEEE 802.11 (Wi-Fi). Надёжнее, чем фильтр по имени или `PhysicalMediaType`.
`Status -eq 'Up'` — только подключённый адаптер.
Если адаптер не найден — выводится список всех интерфейсов и скрипт завершается.
Хардкодить имя или индекс не нужно — они могут отличаться на разных машинах.

```powershell
Write-Host "Adapter: $($adapter.Name)"
```
`$($adapter.Name)` — подстановка свойства объекта в строку. PowerShell работает с кириллицей нативно, `chcp` не нужен.

```powershell
if ($c -eq "1") {
    netsh interface ipv4 set address name="$($adapter.Name)" source=dhcp
    netsh interface ipv4 set dns    name="$($adapter.Name)" source=dhcp
}
```
Переключение на DHCP. `netsh` вызывается из PowerShell напрямую — так же надёжно, как из CMD.
`name="$($adapter.Name)"` — передаём имя адаптера, полученное через `Get-NetAdapter`.

```powershell
if ($c -eq "2") {
    netsh interface ipv4 set address name="$($adapter.Name)" static 192.168.1.50 255.255.255.0 192.168.1.1
    netsh interface ipv4 set dns    name="$($adapter.Name)" static 8.8.8.8
}
```
Установка статического IP. Адрес, маска и шлюз — в одной строке. DNS — отдельно.

```powershell
if ($c -eq "3") {
    Write-Host "Model:       $($adapter.InterfaceDescription)"
    Write-Host "Link:        $($adapter.Status)"
    Write-Host "Speed:       $($adapter.LinkSpeed)"
    Write-Host "Full Duplex: $($adapter.FullDuplex)"
}
```
Вывод информации об адаптере — только чтение, ничего не меняет.
- `InterfaceDescription` — полное название модели сетевой карты
- `Status` — физическое состояние линка: `Up` (кабель/сигнал есть) или `Down`
- `LinkSpeed` — текущая скорость соединения (например, `433.3 Mbps`)
- `FullDuplex` — `True` = полный дуплекс (одновременная передача в обе стороны)

Все эти свойства возвращает объект `Get-NetAdapter`, который уже был получен ранее для автодетекта.

## Разница между .bat и .ps1

| Аспект | bat | PowerShell |
|--------|-----|------------|
| Аспект | bat | PowerShell |
|--------|-----|------------|
| Среда | CMD.exe | PowerShell |
| Поиск адаптера | `powershell` (вызов из CMD) | `Get-NetAdapter` нативно |
| Конфигурация IP/DNS | `netsh` | `netsh` |
| Инфо об адаптере | Нет | Опция 3: модель, линк, скорость, дуплекс |
| Кодировка | `chcp 1251` обязателен | Кириллица работает нативно |
| Переменные | `%VAR%` | `$var` / `$($obj.Prop)` |

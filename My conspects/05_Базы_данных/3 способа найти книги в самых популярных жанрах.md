## Задание
Вывести информацию о книгах (название книги, фамилию и инициалы автора, название жанра, цену и количество экземпляров книги), написанных в самых популярных жанрах, в отсортированном в алфавитном порядке по названию книг виде. Самым популярным считать жанр, общее количество экземпляров книг которого на складе максимально.

### **Способ 1: JOIN подзапросов в WHERE**

```sql
SELECT title, name_author, name_genre, price, amount
FROM book
JOIN author USING(author_id)
JOIN genre USING(genre_id)
WHERE genre_id IN (
    -- Подзапрос для поиска жанров с макс. количеством
    SELECT genre_id FROM
    (
        -- 1. Считаем сумму по каждому жанру
        SELECT genre_id, SUM(amount) sum_amount
        FROM book
        GROUP BY genre_id
    ) genre_data
    JOIN
    (
        -- 2. Находим максимальную сумму
        SELECT SUM(amount) sum_amount
        FROM book
        GROUP BY genre_id
        ORDER BY sum_amount DESC  -- DESC для максимума!
        LIMIT 1
    ) max_genre_amount
    USING(sum_amount)  -- Соединяем по сумме
)
ORDER BY title ASC;
```

**📌 Ключевые моменты:**
- Два подзапроса соединяются через `JOIN USING(sum_amount)`
- `ORDER BY sum_amount DESC` обязателен для получения **максимума**
- `IN` вместо `=` на случай нескольких жанров с одинаковой суммой
- Сложный для чтения из-за двойной вложенности

### **Способ 2: HAVING + подзапрос**

```sql

SELECT title, name_author, name_genre, price, amount
FROM book
JOIN author USING(author_id)
JOIN genre USING(genre_id)
WHERE genre_id IN (
    SELECT genre_id
    FROM book
    GROUP BY genre_id
    HAVING SUM(amount) = (
        -- Подзапрос находит максимальную сумму
        SELECT SUM(amount) sum_amount
        FROM book
        GROUP BY genre_id
        ORDER BY sum_amount DESC  -- DESC для максимума!
        LIMIT 1
    )
)
ORDER BY title ASC;
```


**📌 Ключевые моменты:**
- Использует `HAVING` для фильтрации групп по условию
- Более логичная структура: "найди жанры, у которых сумма равна максимальной"
- Меньше вложенности, чем в способе 1
- Подзапрос в `HAVING` выполняется для каждой группы

### **Способ 3: WITH (CTE)**

```sql

WITH 
-- 1. Считаем общее количество экземпляров по каждому жанру
genre_totals AS (
    SELECT genre_id, SUM(amount) as total_amount
    FROM book
    GROUP BY genre_id
),
-- 2. Находим максимальное значение среди всех жанров
max_total AS (
    SELECT MAX(total_amount) as max_amount
    FROM genre_totals
),
-- 3. Получаем ID жанров с максимальным количеством
popular_genres AS (
    SELECT genre_id
    FROM genre_totals, max_total  -- CROSS JOIN
    WHERE total_amount = max_amount
)
-- 4. Основной запрос
SELECT title, name_author, name_genre, price, amount
FROM book
JOIN author USING(author_id)
JOIN genre USING(genre_id)
WHERE genre_id IN (SELECT genre_id FROM popular_genres)
ORDER BY title ASC;
```

**📌 Ключевые моменты:**
- **Пошаговая логика:** каждый CTE выполняет одну задачу
- **Максимальная читаемость:** запрос читается сверху вниз
- **Повторное использование:** результат `genre_totals` используется дважды
- **Легко отлаживать:** можно проверить каждый CTE отдельно
- `CROSS JOIN` между `genre_totals` и `max_total` создает декартово произведение (безопасно, т.к. max_total содержит 1 строку)

### **Сравнительная таблица**

| Критерий                           | Способ 1 (JOIN) | Способ 2 (HAVING) | Способ 3 (WITH) |
| ---------------------------------- | --------------- | ----------------- | --------------- |
| **Читаемость**                     | Низкая          | Средняя           | **Высокая**     |
| **Производительность**             | Средняя         | Средняя           | **Высокая**     |
| **Сложность понимания**            | Высокая         | Средняя           | **Низкая**      |
| **Количество уровней вложенности** | 3               | 2                 | **1 (плоский)** |
| **Возможность отладки частей**     | Нет             | Нет               | **Да**          |

### **Рекомендации:**
- **Для простых запросов:** используйте способ 2 (HAVING)
- **Для сложных/рабочих запросов:** используйте способ 3 (WITH)
- **Избегайте** способа 1 в реальных проектах из-за низкой читаемости
